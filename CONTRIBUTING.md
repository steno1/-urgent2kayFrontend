# Contributing to URGENT 2KAY 🏦

Thank you for contributing to this project! Please follow these guidelines to ensure smooth collaboration.

## 🚀 How to Contribute

1. **Create a Branch**
   - Always branch off from `dev`
   - Use a meaningful name like: `feature/payment-api` or `fix/login-error`

2. **Push Your Work**
   - Push your changes to your branch regularly.

3. **Open a Pull Request**
   - PRs should be opened **into `dev`**.
   - Add a clear title and description of what you did.

4. **Assign a Reviewer**
   - Assign your closest teammate to review first
- Final merge approval is by **@steno1 (Princeley Onu)**

5. **Wait for Review**
   - Do not merge your own PR.
   - Only  **@steno1 (Princeley Onu)** can merge to `dev` or `main`.

6. **Resolve Comments**
   - If any changes are requested, make them and push again.

7. **Keep It Clean**
   - Format your code and remove unused imports or comments.

---

## ✅ Code Guidelines

- Use **TypeScript** syntax properly.
- Use **Tailwind CSS** only for styling.
- Folder structure must be followed.
- Reusable components should go in `src/components`.

---

## 🤝 Communication

- Always communicate major ideas or blockers in the group.
- Ask before introducing any new libraries.

Thanks for helping us make URGENT 2KAY better! 💪
